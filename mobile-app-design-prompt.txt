==================================================
MOBILE PRODUCT DECISIONS — TREAT AS REQUIREMENTS
==================================================

Mobile IA must be:

1. Top app bar
- logo
- city selector
- hamburger menu
- no dedicated refresh button in the bar; refresh uses standard pull-to-refresh/browser refresh behavior

2. Fixed top TrafficNOW! area
- swipeable route cards
- this is the main control in mobile
- keep full route details visible at all times:
  - route name
  - origin
  - destination
  - qualitative traffic status
  - current speed
  - usual range
  - weather
  - AQI / air quality
- adjacent cards may peek to indicate swiping
- each swipe changes the selected route

3. Main content below the fixed route cards
- route-bound Question
- verdict chart
- speed/trip duration chart

4. Mobile simplifications
- NO Time Travel
- NO baseline selector
- NO auto-refresh
- NO full desktop control density
- baseline and recent periods are predetermined in mobile
- swiping routes updates the Question, verdict, and charts
- mobile analytics remain route-specific but simplified

5. Loading behavior
- use branded full-screen car animation on:
  - initial page load
  - manual refresh/page refresh
- avoid background auto-refresh churn

6. Sharing
- support native share behavior on mobile using browser capabilities where available
- use Web Share API with graceful fallback

7. Chart presentation
- verdict chart should feel visually open
- remove unnecessary card border if present
- use maximum horizontal space
- allow the line plot to extend close to screen edges
- supporting values can appear as compact legend/labels below
